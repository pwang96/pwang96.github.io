Audio Visualizer

I created this web app with a friend because we both love music. While there are many different audio visualizers out there, we wanted to make a unique, interactive music visualizer. Our 3D visualizer is implemented only using Javascript (and jQuery). In particular, we used three.js for the animations. Much like other visualizers, we used the Fast Fourier Transform of the song to drive the animations. The size of the spheres depends on the amplitude of the FFT, whereas the color of the spheres depend on the overall "flavor" of the song, which also depends on which frequencies are present.

League of Legends Champion Recommender

This web application uses machine learning (scikit-learn) to predict which League of Legends champions a player would enjoy playing, based on playstyle history. League of Legends, created by Riot Games is the most played PC game in North America and Europe, where players control a different champion every game. Currently there are 133 champions (new champions are being created every few months).  This web app is powered entirely by the Django web framework for Python. When a username is entered into the site, the Riot API to get stats about the player, some of which I display on the results page. The data is parsed, and then I find a few hundred players around the same ELO as the user. I create a matrix for each player that describes their champion playstyle, and I use cosine similarity to determine which players are most similar.  Afterwards, I look for common champions in the players who are most similar, and I recommend three of them.  I created this app because I'm a big fan of the game, but also because I wanted to get some experience with full stack web development.

MEASURE

❮ ❯

During my time as an intern at NIST, I created a program called MEASURE. MEASURE is a program written in Python that automates and simplifies the mass calibration process. MEASURE was started by Alexander Moses and continued by Peter Wang. The mass calibration process is a lengthy process that NIST follows to calibrate customer weights, using NIST’s internal weights, which are traceable back to the International Prototype Kilogram (IPK) in Paris.  MEASURE provides a GUI (graphical user interface) that allows the metrologists to conduct a calibration much more easily. MEASURE also includes a MySql database interface, where comparator information, thermometer info, NIST weight info, customer weight info, and much more, are all stored. MEASURE allows metrologists to easily pick which weights they want to calibrate, how they want to calibrate them, and on which balance the weights should be calibrated on. Once all the settings are chosen, MEASURE will then communicate with the comparator through a COM port and automatically compare the weights with each other. At the same time, MEASURE communicates with the thermometer, hygrometer, and barometer to get environmental readings at set intervals.  Upon finishing, an output file will automatically be generated.
